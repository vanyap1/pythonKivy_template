# Телеметрійний інтерфейс Smart Home System

## Що реалізовано

Створено повний інтерфейс моніторингу телеметрії системи Smart Home з відображенням всіх даних з `GatewayDto`.

### Компоненти інтерфейсу (екран 1024x600)

#### 1. **Ліва панель (200px)** - Аналогові вимірювачі
- **Вольтметр** - напруга батареї (0-50V)
- **Амперметр** - струм батареї (-50 до +50A)
- **Термометр** - температура насоса циркуляції (0-80°C)

#### 2. **Центральна панель (550px)** - Графік температур
Графік відображає 5 температурних кривих в реальному часі:
- 🟠 **Насос циркуляції** (`mainCirkulationPumpTemp`) - помаранчевий
- 🟢 **Бойлер** (`boilerTemperature`) - зелений
- 🟡 **Сонячний колектор** (`solarCollectorTemp`) - жовтий
- 🔵 **Теплоносій** (`coolantTemperature`) - блакитний
- 🟣 **Котел** (`kotelActTemp`) - фіолетовий

Діапазон: -20°C до +100°C
Історія: до 1000 точок

#### 3. **Права панель (254px)** - Інформаційні панелі

##### 🔋 UPS (120px висота)
- Напруга батареї
- Струм (заряд/розряд)
- Енергія (Wh)
- Джерело живлення (AC🟢/DC🔴)
- Стан системи

##### 🔥 КОТЕЛ (135px висота)
- Статус (Вимкн/Працює/Очікує/Помилка)
- Поточна температура
- Верхня/нижня межі (Hi/Lo)
- Температури запуску/зупинки (Run/Stop)
- Гістерезис (Δ)

##### 💧 ВОДА (125px висота)
- Тиск води (mBar)
- Нижня межа тиску та дельта
- Рівень води (%)
- Цифрові входи/виходи водного контролера

##### 🌡️ СЕНСОРИ (125px висота)
- Струм насоса (mA) та режим роботи
- Рівень освітлення
- Допоміжний ADC канал
- Стан цифрових входів/виходів

### Дизайн

- **Фон панелей**: напівпрозорий темно-сірий з заокругленими кутами
- **Обрамлення**: кольорові рамки відповідно до типу даних:
  - Блакитний (UPS)
  - Помаранчевий (Котел)
  - Синій (Вода)
  - Зелений (Сенсори)
- **Текст**: світло-сірий на темному фоні для зручності читання
- **Заголовки**: кольорові, жирні, з декоративними лініями

### Оновлення даних

Всі дані автоматично оновлюються **щосекунди** через:
- `update_time()` - загальний таймер оновлення
- `temperatureGraphUpdate()` - оновлення графіків
- `updateTelemetryDisplay()` - оновлення всіх текстових даних

### Джерело даних

Дані отримуються через UDP від Smart Home Gateway:
- IP: `192.168.1.18`
- RX порт: `4030`
- TX порт: `4031`

CAN повідомлення автоматично парсяться через `GatewayDto.canMsgParse()`.

## Функції для розширення

### Додавання нового параметру на панель

```python
# 1. Створити Label
self.newParam = Label(text='Назва: ---', font_size=11, color=rgb('DDDDDD'),
                     size_hint_y=None, height=18, halign='left', padding=(10, 0))
self.newParam.text_size = (244, None)

# 2. Додати до відповідного Box
self.sectionBox.add_widget(self.newParam)

# 3. Оновити в updateTelemetryDisplay()
self.newParam.text = f'Назва: {self.gateway.newValue}'
```

### Додавання нової температури на графік

```python
# 1. Створити масив точок в __init__
self.newTempPoints = []

# 2. Додати криву на графік
self.temperatureGraph.add_yPlot(self.newTempPoints, 'FF0000', label='Нова(°C)')

# 3. Оновити в temperatureGraphUpdate()
self.newTempPoints.append(float(self.gateway.newTemperature))
if len(self.newTempPoints) > 1000:
    self.newTempPoints.pop(0)
```

## Відомі обмеження

- Максимум 1000 точок на графіку (близько 17 хвилин історії при оновленні щосекунди)
- Розмір екрану жорстко закодований під 1024x600
- Деякі назви статусів можуть потребувати локалізації
